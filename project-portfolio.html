Case Study

Designing and Building
My Portfolio

A self-initiated project built to create an always-available, online portfolio — while strengthening my web design capability through hands-on HTML and CSS.

HTML CSS Figma Photoshop AI-assisted workflow

Context

This portfolio was created to present my work in a more structured and accessible way — not only as finished visuals, but as communication solutions developed within real business environments.

A key motivation was the need for an always-available, online portfolio that could be shared easily and experienced interactively, rather than existing only as static PDFs.

As part of strengthening my web design capability, I worked more intentionally with HTML and CSS. Building this site became both a practical solution and an opportunity to apply and test that knowledge in a real project environment.

Rather than relying on template platforms, the site was built from scratch to maintain full control over layout, hierarchy, and interaction.

The Problem

Traditional portfolio formats often present work as isolated visuals without context, process, or reasoning. I needed a platform that could communicate thinking, structure projects clearly, remain accessible online at all times, and reflect real-world corporate communication environments.

Balancing clarity, usability, and storytelling became the central challenge.

Objective

  • Present work in structured categories (Print, Web, Digital)
  • Demonstrate design thinking and decision-making — not only visuals
  • Apply working knowledge of HTML and CSS
  • Keep the experience minimal, fast, and distraction-free
  • Reflect professional communication design experience
  • Serve as a foundation for continued UX/UI growth

My Role

  • Information architecture and content structure
  • Visual layout and UI decisions
  • Case study writing and narrative development
  • Image preparation and mockup creation
  • HTML and CSS implementation
  • Gallery and modal interaction structure
  • Navigation logic and anchor linking
  • Browser testing and layout refinement

Tools Used

  • Figma — wireframing and layout planning
  • Adobe Photoshop — image preparation and mockups
  • Adobe InDesign — print layout references
  • HTML — structure and page composition
  • CSS — layout, spacing, type, interaction styling
  • CodePen — development and iteration environment
  • Browser developer tools — layout testing and responsiveness

AI in the workflow

AI-assisted tools were used to support ideation, structural thinking, copy refinement, and technical troubleshooting. This approach draws from formal AI training and ongoing exploration of how emerging tools can support design workflows responsibly and efficiently. AI was used as a support mechanism — not a replacement for design thinking or execution.

Process

The process began with structuring information before designing visuals. I mapped navigation flow, defined project categories, and built reusable layout patterns to maintain consistency.

The gallery approach keeps browsing focused, while modal viewing preserves context so viewers can engage with work without being bounced around the site.

01

Structure first

Mapped sections, anchors, and hierarchy before styling.

02

System design

03

Iteration + testing

Key Design Decisions

  • Used large category panels instead of dense thumbnail grids to reduce clutter
  • Implemented a lightbox gallery to maintain context while viewing projects
  • Separated process thinking from visual output for clarity
  • Kept the palette neutral so the work remains the focal point
  • Introduced human presence in the About section to add warmth and approachability
  • Designed the structure to scale as new projects are added

Outcome

The result is a portfolio that communicates both design execution and structured thinking. It presents work within context, demonstrates technical initiative, and reflects a disciplined approach to visual communication.

The site also serves as a foundation for continued growth into more interactive and UX-informed digital work.

What I Learned

  • Clarity beats decoration
  • Hierarchy beats visual density
  • Systems beat one-off layouts
  • Usability beats aesthetic complexity

What I’d Improve Next

  • Accessibility enhancements
  • Performance optimisation (image sizing and loading)
  • More refined mobile responsiveness
  • Micro-interactions where they add clarity (not noise)
  • Expanded case studies with clearer measurable impact
